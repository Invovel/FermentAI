## 🎯 一句话

FermentAI 是**三个子系统协同工作** — 自动搜索与数据采集、知识发酵与存储、多Agent编排调度 — 给你一个**完全本地运行、自我进化的AI实验室**。

## 🧩 系统构成

FermentAI 不是一个单点工具，而是三个子系统协同：

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│   Research-Skill              LLM-Wiki              Agent-team   │
│   ┌──────────────┐       ┌──────────────┐       ┌─────────────┐ │
│   │   数据管道    │  ──→  │   发酵引擎    │  ──→  │   编排调度   │ │
│   │              │       │              │       │             │ │
│   │ arXiv搜索    │       │ 知识图谱      │       │ Claude      │ │
│   │ GitHub扫描   │       │ 热值/衰减     │       │ GPT         │ │
│   │ 论文入库     │       │ 混合检索      │       │ 本地LLM     │ │
│   │ 自动下载     │       │ 关联推理      │       │ 动态路由    │ │
│   └──────────────┘       └──────────────┘       └─────────────┘ │
│         ↑                      ↑                      ↑         │
│         └──────────────────────┼──────────────────────┘         │
│                      发酵引擎 (Fermentation Engine)              │
│                (连接、加权、进化所有数据)                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

| 子系统 | 角色 | 比喻 |
|--------|------|------|
| **Research-Skill** | 自动数据采集 | **双手** — 搜索、下载、入库 |
| **LLM-Wiki** | 知识发酵与存储 | **记忆** — 存储、连接、排序、衰减 |
| **Agent-team** | 多Agent编排调度 | **大脑** — 路由任务、学习模式、协同工作 |

**发酵引擎串联三者** — 数据流入，关联形成，热值涨落，Agent互相学习。

## 🔬 核心创新：知识发酵

> 原始数据 → 使用痕迹 → 自动关联 → 持续进化的智能

```
传统工具:
  搜索 → 阅读 → 遗忘（数据是死的）

FermentAI:
  Research-Skill 找论文 → LLM-Wiki 存储并建立关联
  → 你发查询 → Agent-team 路由到最佳Agent → 生成答案
  → 痕迹记录 → 高频主题自动触发新一轮搜索
  → 知识图谱更新权重 → 一切都在变聪明
  → ↓
  下一次查询受益于之前发生的一切
```

## 🔄 完整运转流程

```
第1天: Research-Skill 搜索arXiv → 找到5篇论文 → 自动下载 → 提取全文
第2天: LLM-Wiki 入库 → 分块 → 嵌入 → 存入知识库 + 构建关联图
第3天: 你提问"RAG在蛋白质设计中的应用"
第3天: Agent-team 路由 → Claude（生物信息学最佳）→ 生成答案
第3天: 发酵引擎记录：主题=RAG+蛋白质, 使用文献3篇, 置信度=0.85
第5天: 同一主题被再次查询 → 热值 +0.05
第7天: 主题点击阈值 → 触发器触发 → Research-Skill 再次搜索
第10天: 3篇新论文自动下载、关联、推送通知
同期: Agent-team 发现GPT在蛋白质折叠任务反复失败
      → 更新Agent画像 → 未来的蛋白质任务自动路由给Claude
```

## ⚡ 快速开始

```bash
# 克隆仓库
git clone https://github.com/Invovel/FermentAI.git && cd FermentAI

# 安装依赖
pip install -r requirements.txt

# 拉取模型 (Ollama)
ollama pull nomic-embed-text    # 嵌入模型
ollama pull gemma4:e2b          # 生成模型

# 用你的Obsidian知识库初始化
set FERMENT_VAULT_PATH=D:\Obsidian\Hub\WikiVault
python fermentai build

# 启动API服务
python fermentai serve
```

```python
# 查询你的领域AI实验室
import requests
r = requests.post("http://localhost:8000/query", json={
    "query": "扩散模型在蛋白质设计中的最新进展",
    "top_k": 5,
    "use_hyde": True,
    "use_rerank": True,
    "prefer_agent": "auto"    # 让Agent-team自动选择
})
print(r.json()["answer"])
```

## 🎯 使用场景

### 实验室 AI 生态
> "我们实验室用一台服务器跑 FermentAI。Research-Skill 每天自动入库最新论文。发酵引擎把它们和已有知识连接。Agent-team 把复杂生物信息查询路由给 Claude，简单查询走本地 LLM。新人直接查询整个实验室的知识历史。"

### 个人研究加速器
> "我接入了我的 Obsidian 知识库。FermentAI 发现我反复查询蛋白质语言模型——自动下载了3篇新论文，关联到我的笔记，Agent-team 还根据查询类型推荐用哪个Agent。知识在帮我主动生长。"

### 企业知识平台
> "公司文档、代码仓库、Slack 讨论 → 全部喂给 FermentAI。发酵引擎把它们连成图谱。客服 Agent 带溯源精准检索。开发 Agent 复用验证过的代码。系统学习哪个Agent擅长哪类任务，越用越高效。"
